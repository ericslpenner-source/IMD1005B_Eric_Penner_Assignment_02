My client is a freelance photographer who takes jobs taking pictures of people, pets, and events/locations for commemoration or staging.<br>
The site serves as an online portfolio to attract potential clients.<br>
Based on the feedback from Assignment 2, I removed the 'main' tag, added a 'header' tag, added an h1 on the landing page/changed the h3 on the other pages to h1, removed the commented-out code blocks, and added alts on the images.<br>
I did not use AI at any point in the process of creating this website.<br>
<br>
Based on the feature requests, I have added:<br>
1 - RESPONSIVE HAMBURGER MENU: The navbar on each page now includes a hamburger menu at lower screen-widths. I had some issues formatting the hamburger menu once clicked.<br>
2 - FORM VALIDATION: The contact form now validates that your message contains a first and last name, as well as a valid email address.<br>
3 - BACK TO TOP BUTTON: Each page now has a "back to top" button that appears when the user has navigated to the bottom. I had some issues getting it to hide itself when the page was being loaded, because I had two display: statements in the css.<br>
4 - DYNAMIC CONTENT RENDERING: The rates table on the "contact & rates" page (contact.html) is now created dynamically from an object array in ratesTable.js.<br>
5 - ACCORDION FAQ: The "about us" page (our-story.html) now has an accordion FAQ below the artist bio.<br>
6 - LIVE FILTERING: Not added.<br>
7 - DISPLAY API DATA: The new "portfolio" page (portfolio.html) now displays random pictures of dogs sourced from https://dog.ceo/dog-api/. I ran into issues getting the images to display properly initially because I was using the wrong fetch structure (only (data) with no (response)).<br>
